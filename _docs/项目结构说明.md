# 项目结构说明（当前状态）

这个文档描述 HoCloth 现阶段的实际结构，而不是早期设想。

## 现有分层

- `authoring/`：Blender 侧 UI、面板、操作与导出
- `components/`：资源注册、属性定义和组件组织
- `runtime/`：运行时会话、输入整理、桥接和姿态应用
- `_native/`：C++ 侧运行时与碰撞世界
- `compile/`：预构建与编译辅助
- `_docs/`：设计说明、进度记录与踩坑总结

## 当前结构判断

- 当前主线已经从“通用 XPBD 重构”转向“MC2 BoneSpring 数据流对齐”
- runtime 的职责正在从“直接驱动碰撞求解”转向“固定步调度 + 数据分层”
- 旧的接触 lambda 路线不再是主线，只保留作为历史记录

## 已确认的工作重点

- 先把 `BoneSpring` 状态拆清楚
- 先把固定步调度稳定下来
- 先把碰撞体处理改成局部投影
- 再扩展到更完整的骨链和衣物链

## 目前的风险点

- 文档里仍可能残留“XPBD contact lambda 主线”的旧表述
- 部分命名还没完全统一，例如 `base_pos` / `next_pos` / `old_pos` / `velocity_pos`
- 碰撞与约束的执行顺序还在收敛中
