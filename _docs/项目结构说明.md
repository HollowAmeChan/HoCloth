# HoCloth 项目结构说明

本文档说明当前仓库的实现分层，重点是 `MC2` 对齐的 `BoneChain` 主线，以及 `Bonex` 风格的 Blender 交互。

## 分层

### Blender 侧
- `authoring/`：编辑、提取、交互工具
- `components/`：组件定义与持久化数据
- `compile/`：把 authoring 数据编译为 runtime 可用数据
- `runtime/`：会话、桥接、回写

### Native 侧
- `_native/scene/`：scene 生命周期
- `_native/solver/`：固定步进调度和约束求解
- `_native/constraints/`：约束实现
- `_native/collision/`：碰撞处理
- `_native/mapping/`：网格或骨结果映射
- `_native/binding/`：Python / C++ 桥接

## 当前推进重点

1. `BoneChain` 参数与行为继续对齐 `Magica Cloth 2`
2. Blender UI 与工作流继续靠近 `Bonex`
3. 先把 `compile -> build -> step -> pose apply` 闭环稳定住
4. 为 `ClothBone`、`MeshCloth` 预留可扩展的数据结构
