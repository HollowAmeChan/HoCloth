## 整体结构

仓库根目录本身就是 Blender 插件目录。
本地开发时，Blender 可以直接把当前仓库作为插件目录加载，因为根目录下就有 `__init__.py`。

## 运行时相关目录

`authoring/`
: Blender 侧的操作器、提取逻辑和界面入口。

`compile/`
: 把 authoring 数据转换为运行时可消费的编译结果。

`components/`
: Blender 侧属性组、注册表和数据持久化相关代码。

`runtime/`
: Python 侧运行时桥接、session 管理和姿态回写逻辑。

`_bin/`
: Blender 运行时实际会加载的二进制内容，包括 native 模块和 PhysX DLL。

## 开发期目录

`_native/`
: C++ 源码、头文件和 CMake 配置，负责 PhysX 支撑的运行时实现。
发布包中不会包含这个目录。

`_third_party/`
: 外部依赖目录，例如 PhysX 及其相关构建内容。

`_docs/`
: 架构说明、流程说明和打包文档。

`_build/`
: 本地 staging、预发布整理和临时构建输出目录。

`_dist/`
: 最终发布 zip 输出目录。

## 根目录关键文件

`__init__.py`
: Blender 插件入口文件。

`build_physx.ps1`
: 当 PhysX 第三方产物缺失或需要重建时使用的辅助脚本。

`package_addon.ps1`
: 本地打包脚本，可选在打包前自动构建 native 模块，并输出干净的发布包。

`CMakePresets.json`
: 当前本地 native 构建使用的预设配置，其中包括已经验证可用的 `vs2022-release-physx` 流程。
