# HoCloth 项目结构说明

这份文档说明当前仓库在清理旧物理依赖之后的目录职责。当前项目目标已经调整为“Blender 插件 + 原生占位 runtime + 后续自研 XPBD 接入”。

相关文档：

- [发布流程与构建说明.md](./发布流程与构建说明.md)

## 整体结构

仓库根目录本身就是 Blender 插件目录，原因是根目录直接包含 `__init__.py`。开发时可以把整个仓库直接作为插件目录使用。

## 运行时目录

`authoring/`
: Blender 侧的提取逻辑、操作器和面板入口。

`components/`
: Blender 侧属性组、组件注册表和持久化结构。

`compile/`
: 把 authoring 数据整理为稳定的 compiled scene 数据。

`runtime/`
: Python 侧 runtime bridge、session 管理和姿态回写逻辑。

`_bin/`
: Blender 运行时真正会加载的二进制目录。现在只应该放 `hocloth_native` 一类原生模块。

## 开发目录

`_native/`
: C++ 源码、头文件和 CMake 配置。当前实现是 XPBD 方向的占位 runtime，用来维持编译链和 Python 接口稳定。

`_third_party/`
: 第三方源码依赖目录。当前主要保留 `nanobind` 等构建相关依赖。

`_docs/`
: 项目结构、构建流程和发布流程文档。

`_build/`
: 本地调试输出、运行时调试导出和 staging 目录。

`_dist/`
: 本地打包生成的 zip 输出目录。

## 根目录关键文件

`__init__.py`
: Blender 插件入口。

`package_addon.ps1`
: 打包脚本，可选在打包前自动执行 native 构建。

`CMakeLists.txt`
: 仓库级 CMake 入口，转入 `_native/` 子工程。

`CMakePresets.json`
: 当前推荐的本地 native 构建预设，默认走 `vs2022-release-native`。

`工程大纲.md`
: 当前项目蓝图，已经切换为“自研 XPBD 路线”。
