# HoCloth 目录功能说明

本文档面向当前“对齐 `Magica Cloth 2` 物理内核与 `Bonex` 交互”的仓库状态。

## 顶层目录

### `.github/`
用于存放 GitHub Actions 工作流。

### `_docs/`
用于存放项目结构、构建流程和实现记录文档，不参与发布包。

### `_native/`
用于存放原生模块源码。当前职责是：
- 作为 `MC2` 对齐实现的 native runtime
- 保持 Python 与 C++ 的桥接接口稳定
- 为后续更深入的 `BoneChain`、`ClothBone`、`MeshCloth` 实现预留入口

### `_third_party/`
用于存放构建相关第三方依赖。

### `_bin/`
Blender 真正会加载的本地二进制目录。当前期望内容是：
- native 扩展模块
- 运行时所需的最小依赖

### `_build/`
用于存放本地构建输出、调试中间文件和 staging 目录。

### `_dist/`
用于存放最终打包生成的 zip 文件。

## 业务目录

### `authoring/`
Blender 侧作者工作流与数据提取逻辑。

### `components/`
组件定义、注册与持久化结构。

### `compile/`
把 Blender 侧 authoring 数据编译成稳定的中间数据，供 runtime 使用。

### `runtime/`
Python 侧 runtime 会话管理、native bridge 与姿态回写逻辑。

## 关键文件

### `package_addon.ps1`
本地与发布阶段共用的打包脚本。

### `CMakeLists.txt`
仓库级入口，用于把构建导向 `_native/`。

### `CMakePresets.json`
本地构建预设。

### `工程大纲.md`
项目蓝图文档，当前以 `MC2` 对齐路线为主。
