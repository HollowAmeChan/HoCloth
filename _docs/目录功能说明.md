# HoCloth 目录功能说明

本文档面向当前“保留占位 native runtime、准备自研 XPBD”的仓库状态。

相关文档：

## 顶层目录

### `.github/`

用于存放 GitHub Actions 工作流。

### `_docs/`

用于存放项目结构、构建流程和发布说明文档。它不参与发布包。

### `_native/`

用于存放原生模块源码。当前职责是：
- 保持 Python 和 C++ 的桥接接口稳定
- 提供一个不依赖第三方大型物理 SDK 的占位 runtime
- 为后续自研 XPBD 解算器保留接入点

### `_third_party/`

用于存放构建相关第三方依赖。目前应只保留与构建确实相关的依赖，例如 `nanobind`。

### `_bin/`

这是 Blender 真正会加载的本地二进制目录。当前期望内容：
- `hocloth_native` Python 扩展模块

不应再出现旧版物理 SDK 的 `.dll` 或对应构建缓存。

### `_build/`

用于存放：
- 运行时调试输出
- staging 目录
- 本地临时构建辅助文件

### `_dist/`

用于存放最终打包出的 zip 文件。

## 业务目录

### `authoring/`

Blender 侧作者工作流与数据提取逻辑。

### `components/`

组件定义、注册与持久化结构。

### `compile/`

把 Blender 侧 authoring 数据编译成稳定的中间数据结构，供 runtime 使用。

### `runtime/`

Python 侧 runtime 会话管理、native bridge 和姿态回写逻辑。native 模块缺失时也可以退回 Python 占位 backend。

## 关键文件

### `package_addon.ps1`

当前本地与发布阶段共用的打包脚本。可选先构建 native，再组装出干净的 Blender 插件目录。

### `CMakeLists.txt`

仓库级入口，负责把构建引导到 `_native/`。

### `CMakePresets.json`

推荐使用的本地构建预设。当前默认是 `vs2022-release-native` 和 `build-vs2022-release-native`。
