# 目录功能说明

本文档描述 HoCloth 当前目录职责。当前主线已经切换为“完整移植 Magica Cloth 2 Core 到 C++ native backend，再接入 Blender”。

## 根目录

```text
HoCloth/
  __init__.py
  authoring/
  components/
  runtime/
  _native/
  _ReferenceProject/
  _third_party/
  _docs/
  _bin/
  _build/
  _dist/
```

## `authoring/`

Blender 侧交互层，负责选择提取、operator、handler、gizmo、工具和视口辅助。

它不负责物理求解，也不应保存 native runtime 内部状态。

## `components/`

Blender 组件与属性组织层，负责把用户可编辑数据组织为 MC2 风格组件容器。

典型内容：

- MagicaCloth BoneSpring / BoneCloth / 后续 MeshCloth 组件
- MagicaSphereCollider / MagicaCapsuleCollider / MagicaPlaneCollider
- wind
- MC2 preset
- UI 可保存参数

## `runtime/`

Blender 与 native backend 的运行时桥接层。

职责：

- authoring snapshot 采样与导出
- session 生命周期
- build / reset / step
- 每帧 transform、dt、风、override 输入
- 骨骼或网格结果回写
- cache / debug dump 调用

## `_native/`

C++ native backend。当前新方向要求它按 MC2 Core 的系统架构重建，而不是继续把 XPBD 逻辑堆在少数 cpp 文件里。

推荐结构见：

```text
_docs/MC2_Core完整移植重构方案.md
```

现有 `_native/src/hocloth_*.cpp` 只作为历史桥接参考，不再作为新物理核心的架构基础。

## `_ReferenceProject/`

参考源码目录。当前最重要的参考是：

```text
_ReferenceProject/MagicaCloth2/Scripts/Core
```

移植时必须优先对照这里的目录和源码文件。

## `_third_party/`

第三方依赖或本地依赖缓存目录。

## `_docs/`

设计说明、推进记录、移植计划、发布说明和历史记录。

当前优先阅读：

```text
工程大纲.md
ai须知.md
_docs/MC2_Core完整移植重构方案.md
```

## `_bin/`

native 构建产物或发布时需要携带的二进制文件。

## `_build/`

本地构建、打包 staging、临时产物目录。

## `_dist/`

发布 zip 等最终分发产物目录。
## Blender / Native 数据交换契约

当前 Python 与 native 的 build / frame / output / debug 数据统一由以下模块约束：

```text
runtime/exchange.py
_docs/Blender_Native数据交换契约.md
```

`runtime/authoring_snapshot.py` 负责产出 MC2 风格 `authoring_snapshot`；`runtime/inputs.py` 负责产出逐帧 frame inputs；`runtime/bridge.py` 负责把 exchange envelope 交给 native。`runtime/compiled_scene.py` 只保留为当前姿态写回和调试导出的 backend scene view，不再是 Blender 侧的重型编译层。
