# HoCloth 本地打包流程

这份文档记录当前仓库已经实际验证通过的一套本地打包流程，目标是从当前工作区直接生成 Blender 可安装的 HoCloth 发布包。

## 1. 什么时候直接打包

如果下面这些条件都满足，可以直接运行 `package_addon.ps1`：

- `_third_party/PhysX/physx/bin/win.x86_64.vc143.mt/release/` 下已经有 PhysX 的 `.lib` 和 `.dll`
- 不需要重建 PhysX 本体
- 你想基于当前工作区的 Python 代码和 `_native/` 代码直接生成发布包

推荐命令：

```powershell
powershell -ExecutionPolicy Bypass -File .\package_addon.ps1 -Version manual-check -IncludeNativeBuild -CreateZip
```

## 2. 什么时候先跑 build_physx

如果遇到下面这些情况，先执行 `build_physx.ps1`：

- PhysX 的 `.lib` / `.dll` 缺失
- 清理过 `_third_party/PhysX/physx/bin/`
- 清理过 `_third_party/PhysX/physx/compiler/`
- 清理过 `_third_party/PhysX/packman-cache/`
- PhysX 构建状态异常，需要完整重建

命令：

```powershell
powershell -ExecutionPolicy Bypass -File .\build_physx.ps1
```

然后再执行：

```powershell
powershell -ExecutionPolicy Bypass -File .\package_addon.ps1 -Version manual-check -IncludeNativeBuild -CreateZip
```

## 3. package_addon 实际会做什么

在当前仓库里，`-IncludeNativeBuild` 会触发下面这套流程：

1. 使用 `vs2022-release-physx` 做 CMake configure
2. 使用 `build-vs2022-release-physx` 做 CMake build
3. 生成 `_bin/hocloth_native.cp311-win_amd64.pyd`
4. 把 native 模块和 PhysX 运行时 DLL 整理到 `_bin/`
5. 生成 staging 目录 `_build/release-<Version>/HoCloth`
6. 生成 zip 文件 `_dist/HoCloth-<Version>.zip`

## 4. 已验证的产物

按下面命令：

```powershell
powershell -ExecutionPolicy Bypass -File .\package_addon.ps1 -Version manual-check -IncludeNativeBuild -CreateZip
```

已经验证可以生成这些关键文件：

- `_bin/hocloth_native.cp311-win_amd64.pyd`
- `_bin/PhysX_64.dll`
- `_bin/PhysXCommon_64.dll`
- `_bin/PhysXFoundation_64.dll`
- `_bin/PhysXCooking_64.dll`
- `_dist/HoCloth-manual-check.zip`

对应的 staging 目录是：

- `_build/release-manual-check/HoCloth`

## 5. 当前发布包排除项

当前 `package_addon.ps1` 在发布时会排除开发期内容，包括但不限于：

- `_native/`
- `_third_party/`
- `_docs/`
- `_build/`
- `_dist/`
- `build_physx.ps1`
- `package_addon.ps1`

这样发布包更接近纯运行时插件目录，不会把本地构建脚本也一起带进 Blender 安装包。
## 6. native 调试补充

如果这次打包或本地验证是为了排查骨骼链 / PhysX runtime，建议额外遵守下面这条顺序：

1. 先关闭正在使用当前插件目录的 Blender。
2. 再执行 `package_addon.ps1 -IncludeNativeBuild` 或单独的 CMake build。
3. 如果链接阶段报 `LNK1104` 且目标文件是 `_bin/hocloth_native.cp311-win_amd64.pyd`，优先判断为 Blender 占用了 native 模块文件。

推荐命令：

```powershell
powershell -ExecutionPolicy Bypass -File .\package_addon.ps1 `
  -Version manual-check `
  -IncludeNativeBuild `
  -CreateZip `
  -CMakeExecutable "C:\Program Files\Microsoft Visual Studio\2022\Community\Common7\IDE\CommonExtensions\Microsoft\CMake\CMake\bin\cmake.exe"
```

补充说明：

- `package_addon.ps1` 现在会优先尝试自动探测 Visual Studio 自带的 `cmake.exe`
- 如果当前环境对 Windows SDK 或 `AppData\Local\Microsoft SDKs` 的访问有限制，configure / build 可能需要在完整本机权限下执行
