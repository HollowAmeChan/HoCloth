# HoCloth 本地打包流程

当前流程面向当前 XPBD 占位 runtime 状态。目标是从当前工作区直接生成可安装到 Blender 的 HoCloth 插件包。

## 1. 直接打包

如果你只想基于当前代码打一个包含 native 占位 runtime 的插件包，使用：

```powershell
powershell -ExecutionPolicy Bypass -File .\package_addon.ps1 -Version manual-check -IncludeNativeBuild -CreateZip
```

## 2. `package_addon.ps1` 会做什么

启用 `-IncludeNativeBuild` 后，脚本会：

1. 使用 `vs2022-release-native` 做 CMake configure
2. 使用 `build-vs2022-release-native` 做 CMake build
3. 把生成的 `hocloth_native` 输出到 `_bin/`
4. 组装 staging 目录 `_build/release-<Version>/HoCloth`
5. 可选生成 `_dist/HoCloth-<Version>.zip`

## 3. 当前预期产物

- `_bin/hocloth_native.cp311-win_amd64.pyd`
- `_build/release-<Version>/HoCloth`
- `_dist/HoCloth-<Version>.zip`

不应再出现额外的外部物理运行库 DLL。

## 4. 打包排除项

当前打包流程会排除这些开发目录：

- `_native/`
- `_third_party/`
- `_docs/`
- `_build/`
- `_dist/`
- `package_addon.ps1`

## 5. native 调试补充

如果构建 `hocloth_native` 时遇到 `LNK1104`，优先检查 Blender 是否还占用着 `_bin/hocloth_native.cp311-win_amd64.pyd`。

推荐顺序：

1. 先关闭正在使用当前插件目录的 Blender
2. 再执行 `package_addon.ps1 -IncludeNativeBuild`
3. 如有需要，显式传入 Visual Studio 自带的 `cmake.exe`

示例：

```powershell
powershell -ExecutionPolicy Bypass -File .\package_addon.ps1 `
  -Version manual-check `
  -IncludeNativeBuild `
  -CreateZip `
  -CMakeExecutable "C:\Program Files\Microsoft Visual Studio\2022\Community\Common7\IDE\CommonExtensions\Microsoft\CMake\CMake\bin\cmake.exe"
```
