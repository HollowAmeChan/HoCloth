# HoCloth 发布说明

当前发布模型已经切换为“`Blender authoring + native MC2 bootstrap runtime`”。发布时会先整理出一个干净的 Blender 插件目录，再压缩为 zip。

相关文档：
- [本地打包流程.md](./本地打包流程.md)
- [发布流程与构建说明.md](./发布流程与构建说明.md)

## 当前推荐命令

```powershell
powershell -ExecutionPolicy Bypass -File .\package_addon.ps1 -Version manual-check -IncludeNativeBuild -CreateZip
```

## 预期产物

- staging 目录：`_build/release-<Version>/HoCloth`
- 发布 zip：`_dist/HoCloth-<Version>.zip`
- native 模块：`_bin/hocloth_native.cp311-win_amd64.pyd`

## 发布排除项

发布包不会包含这些开发期内容：

- `_native/`
- `_third_party/`
- `_docs/`
- `_build/`
- `_dist/`
- `package_addon.ps1`

## 说明

- 当前 native 已不再定位为“XPBD 占位实现”。
- 当前主线目标是先把 `Magica Cloth 2` 的三种模拟类型逐步复现进 Blender：
  - `BoneSpring`
  - `ClothBone`
  - `MeshCloth`
- 当前已经进入 `MC2 BoneSpring` 的第一轮 native 重构阶段，重点是把关键 C# 逻辑翻译到 C++，并保持 Blender 侧编译链路、实时播放链路和打包链路可用。
- 当前阶段不优先扩展 `MC2` 没有的功能，优先把 `MC2` 本身复现完整。
