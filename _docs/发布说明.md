## 发布模型

仓库根目录在本地开发阶段就是 Blender 插件根目录。
真正发布时，不是直接压缩整个仓库，而是先整理出一个干净的 staging 插件目录，再把这个目录打成 zip。

Blender 运行时需要的文件统一放在 `_bin/` 中。
开发期源码、第三方依赖源码和缓存目录不会进入最终发布包。

## 已验证的本地打包命令

如果 PhysX 输出已经存在，并且你希望基于当前工作区直接生成 zip，可以执行：

```powershell
powershell -ExecutionPolicy Bypass -File .\package_addon.ps1 -Version manual-check -IncludeNativeBuild -CreateZip
```

如果 PhysX 的 `.lib` / `.dll` 缺失，或者第三方构建状态已经过期，先执行：

```powershell
powershell -ExecutionPolicy Bypass -File .\build_physx.ps1
```

然后再执行上面的 `package_addon.ps1` 打包命令。

## 预期产物

- staging 目录：`_build/release-<Version>/HoCloth`
- 发布 zip：`_dist/HoCloth-<Version>.zip`
- native 模块：`_bin/hocloth_native.cp311-win_amd64.pyd`

## 当前发布排除项

当前发布流程会排除这些开发期内容：

- `_native/`
- `_third_party/`
- `_docs/`
- `_build/`
- `_dist/`
- `build_physx.ps1`
- `package_addon.ps1`
