# HoCloth 发布说明

当前发布模型暂时保持“Blender authoring + native backend 打包”的技术路线，但项目主线已经切换为“完整移植 Magica Cloth 2 Core 到 C++ 后端”。

短期内发布包的重点不是新增最终用户功能，而是保证 Blender 侧桥接、native module、打包链路在重构过程中不断线。

## 当前推荐打包命令

```powershell
powershell -ExecutionPolicy Bypass -File .\package_addon.ps1 -Version manual-check -IncludeNativeBuild -CreateZip
```

## 预期产物

```text
_build/release-<Version>/HoCloth
_dist/HoCloth-<Version>.zip
_bin/hocloth_native.cp311-win_amd64.pyd
```

## 发布排除项

发布包不包含这些开发期内容：

- `_native/`
- `_third_party/`
- `_docs/`
- `_build/`
- `_dist/`
- `package_addon.ps1`

## 当前主线变化

### 2026-04-24 到 2026-04-25

前一阶段主要围绕 MC2 BoneSpring 做局部复现，已经跑通过 Blender authoring、compiled scene、native binding、native solver 的基础链路，并积累了若干 BoneSpring 复现记录。

### 当前重构方向

现在不再继续把 BoneSpring 当作独立小求解器修补。C++ 侧将全面转向 MC2 Core 风格后端：

- 先建立 `Define / Utility / NativeCollection / Time / ResultCode` 等基础层
- 再建立 `MagicaManager / TeamManager / TransformManager / SimulationManager / ColliderManager / WindManager / ClothManager / VirtualMeshManager`
- 然后通过同一套 manager 和 constraint pipeline 重新贯通 BoneSpring
- 后续再接入 ClothBone 和 MeshCloth

详细方案见：

```text
_docs/MC2_Core完整移植重构方案.md
```

## 发布注意

重构期间允许 native 内核功能短期回退，但不允许打包链路失效。每次重构都应至少确认：

- Blender 插件可以加载
- native module 可以 import
- build / package 命令仍有明确失败原因或可用产物
- `_docs` 中的当前路线说明保持同步
