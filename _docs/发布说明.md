# HoCloth 发布说明

当前发布模型保持为“Blender authoring + native backend 打包”。项目主线已经转向完整移植 Magica Cloth 2 Core 到 C++ 后端，因此发布链路的职责是：保证 Blender 侧桥接、native module、native smoke test、发布目录和 zip 打包在重构期间不断线。

## 标准流程

所有命令都在插件根目录执行：

```powershell
cd "c:\Users\hhh12\AppData\Roaming\Blender Foundation\Blender\4.5\scripts\addons\HoCloth"
```

### 0. VSCode 左下角“生成”

这属于 **本地开发构建 / CMake Tools 构建流程**。

VSCode 的 CMake Tools 会读取 `CMakePresets.json`，使用当前选择的 configure/build preset 或当前 target，直接执行 CMake build。它和发布脚本共用同一套 CMake 工程，但只负责编译 native 目标，不负责整理发布目录，也不负责创建 zip。

常见产物：

```text
_bin/hocloth_mc2_core_smoke.exe
_bin/hocloth_mc2_core_smoke.pdb
_bin/hocloth_native.cp311-win_amd64.pyd
_bin/hocloth_native.pdb
```

这些文件的含义：

- `hocloth_native.cp311-win_amd64.pyd`：Blender/Python 实际 import 的 native module。
- `hocloth_native.pdb`：native module 调试符号。
- `hocloth_mc2_core_smoke.exe`：MC2 Core C++ smoke test 可执行文件。
- `hocloth_mc2_core_smoke.pdb`：smoke test 调试符号。

这个流程适合日常 C++ 开发、快速编译、断点调试和确认 `_bin` 里的 native 产物是否更新。它不等价于发布流程；正式发布仍应使用下面的 `package_addon.ps1` 流程。

### 1. 编译测试

只做 native 配置、编译和 smoke test，不创建发布 zip：

```powershell
powershell -ExecutionPolicy Bypass -File .\package_addon.ps1 -Version build-check -IncludeNativeBuild -RunNativeSmoke
```

### 2. 发布文件夹

构建 native module，并生成可直接从 Blender 安装/加载的发布目录：

```powershell
powershell -ExecutionPolicy Bypass -File .\package_addon.ps1 -Version manual-check -IncludeNativeBuild -RunNativeSmoke
```

产物：

```text
_build/release-manual-check/HoCloth
_bin/hocloth_native.cp311-win_amd64.pyd
_bin/hocloth_mc2_core_smoke.exe
```

### 3. 发布 zip

构建 native module、运行 smoke test，并生成发布 zip：

```powershell
powershell -ExecutionPolicy Bypass -File .\package_addon.ps1 -Version manual-check -IncludeNativeBuild -RunNativeSmoke -CreateZip
```

产物：

```text
_build/release-manual-check/HoCloth
_dist/HoCloth-manual-check.zip
_bin/hocloth_native.cp311-win_amd64.pyd
```

## Fresh Configure

如果 `_native/build/vs2022-release` 来自旧用户名、旧路径、旧 Python，CMake 可能报 source/build cache 不匹配。此时加 `-FreshConfigure`：

```powershell
powershell -ExecutionPolicy Bypass -File .\package_addon.ps1 -Version manual-check -IncludeNativeBuild -RunNativeSmoke -CreateZip -FreshConfigure
```

## 工具链解析

`package_addon.ps1` 会自动寻找：

- `cmake.exe`：优先 PATH，其次 Visual Studio 2022 常见安装目录，再用 `vswhere.exe` 定位 VS 安装。
- Python 3.11：优先 `-PythonExecutable` / `HOCLOTH_PYTHON_EXECUTABLE` / PATH，其次常见 Python 3.11 和 Blender 4.3-4.5 自带 Python 路径。

需要手动指定时：

```powershell
powershell -ExecutionPolicy Bypass -File .\package_addon.ps1 `
  -Version manual-check `
  -IncludeNativeBuild `
  -RunNativeSmoke `
  -CreateZip `
  -FreshConfigure `
  -CMakeExecutable "D:\Microsoft Visual Studio\2022\Community\Common7\IDE\CommonExtensions\Microsoft\CMake\CMake\bin\cmake.exe" `
  -PythonExecutable "D:\Blender\Blender 4.5\4.5\python\bin\python.exe"
```

## 发布排除项

发布包不包含这些开发期内容：

- `_native/`
- `_third_party/`
- `_docs/`
- `_build/`
- `_dist/`
- `_ReferenceProject/`
- `package_addon.ps1`

## 发布注意

重构期间允许 native 内核功能短期波动，但不允许打包链路失效。每轮较大改动后至少确认：

- native module 可以构建。
- `hocloth_mc2_core_smoke.exe` 可以运行。
- `_bin/hocloth_native.cp311-win_amd64.pyd` 存在。
- 发布目录或发布 zip 能生成。
- `_docs` 中当前路线说明保持同步。
