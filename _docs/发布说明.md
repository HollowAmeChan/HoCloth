# HoCloth 发布说明

当前发布模型已经切换为“原生占位 runtime + 后续自研 XPBD 接入”。发布时会先整理出一个干净的 Blender 插件目录，再压缩为 zip。

相关文档：
- [本地打包流程.md](./本地打包流程.md)
- [发布流程与构建说明.md](./发布流程与构建说明.md)

## 当前推荐命令

```powershell
powershell -ExecutionPolicy Bypass -File .\package_addon.ps1 -Version manual-check -IncludeNativeBuild -CreateZip
```

## 预期产物

- staging 目录：`_build/release-<Version>/HoCloth`
- 发布 zip：`_dist/HoCloth-<Version>.zip`
- native 模块：`_bin/hocloth_native.cp311-win_amd64.pyd`

## 发布排除项

发布包不会包含这些开发期内容：

- `_native/`
- `_third_party/`
- `_docs/`
- `_build/`
- `_dist/`
- `package_addon.ps1`

## 说明

当前 native 只是 XPBD 路线的占位实现，作用是保证接口、打包链和 Blender 侧流程先稳定下来，后续再逐步替换成正式的自研求解器。
