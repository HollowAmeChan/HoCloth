## HoCloth 工程大纲

### 1. 项目定位

- **目标产品**：Blender 内可实时预览、可离线烘焙、可导出缓存的角色二级动力学插件。
- **核心路线**：`Blender/Python authoring + C++ 原生 XPBD runtime + 可重建 compiled scene`。
- **目标对标**：
  - 功能侧对标 `Magica Cloth 2` 的 `SpringBone / BoneCloth / MeshCloth`
  - 数据组织侧借鉴 `UniVRM` 的 `Spring / Joint / ColliderGroup / RuntimeProvider`
  - 求解器结构侧借鉴 `PositionBasedDynamics`
  - Blender 交互与工作流侧重点借鉴 `blender_bonex`
- **明确不做**：
  - 不再绑定大型通用物理 SDK 作为主方案
  - 不依赖 Blender 原生刚体系统作为核心解算
  - 不把 Blender 自定义属性直接当作 runtime 输入结构

---

### 2. 产品主线

#### 2.1 SpringBone

- 目标还原 MC2/VRM 风格的单链弹簧骨。
- 典型对象：头发、尾巴、挂件链、裙摆骨、耳朵骨。
- 关键能力：
  - 单链或多链组件
  - 每 joint 独立参数
  - 支持中心骨、惯性、阻尼、重力、拖拽
  - 支持球/胶囊/平面碰撞
  - 可实时播放预览

#### 2.2 ClothBone

- 目标还原 MC2 的骨驱动布料。
- 本质不是“只有一条链”，而是“多根骨或骨网络的布料化解算 + 回写骨骼”。
- 关键能力：
  - 多 root 骨集合
  - 骨间连接拓扑构建
  - 距离/弯曲/形状保持约束
  - 骨姿态回写
  - 与碰撞体、风场、固定点联动

#### 2.3 MeshCloth

- 目标还原 MC2 的代理网格布料。
- 关键能力：
  - 从渲染网格或低模代理网格构建 simulation mesh
  - 代理网格 reduction / weld / selection / mapping
  - simulation mesh -> render mesh 映射
  - 实时低模预览 + 高模缓存输出

---

### 3. 总体架构原则

- **Authoring / Compiled / Runtime 三态分离**
  - Authoring：Blender 里编辑的组件与参数
  - Compiled：稳定、可导出、可调试的中间数据
  - Runtime：native 持有的 scene / solver / buffer / state
- **结构变更统一走 Rebuild**
  - 组件增删改
  - 骨拓扑变化
  - 碰撞体结构变化
  - 网格拓扑变化
- **每帧只传高频输入**
  - `dt`
  - 根节点/中心骨变换
  - 外力、风、模型级开关
  - 少量 joint/model 覆盖参数
- **求解器只接收扁平化结构**
  - 不向 C++ 传 Blender 对象
  - 不向 Python 暴露底层 solver 内部对象
  - Python 只看到 `handle + arrays + dict-like config`
- **实时主线优先骨回写，离线主线优先缓存**
  - SpringBone / ClothBone 重点是骨结果实时回写
  - MeshCloth 高模结果优先走缓存导出

### 5. 组件系统目标

#### 5.1 一级组件

- `SPRING_BONE`
- `CLOTH_BONE`
- `MESH_CLOTH`
- `COLLIDER`
- `COLLIDER_GROUP`
- `WIND`
- `PIN` / `ATTACHMENT`
- `CACHE_OUTPUT`
- `PRESET`

#### 5.2 组件组织原则

- 保留主组件目录 + 类型容器的双层结构。
- 主组件只负责：
  - `component_id`
  - `component_type`
  - `container_index`
  - `display_name`
  - `enabled`
  - `ui_expanded`
- 类型容器负责：
  - 实际 authoring 数据
  - 类型特有参数
  - 与 Blender 对象的引用关系

#### 5.3 组装关系

- `SPRING_BONE`
  - 引用一组 `joint settings`
  - 关联多个 `COLLIDER_GROUP`
  - 可选 `center object`
- `CLOTH_BONE`
  - 引用多 root 骨
  - 关联多个 `COLLIDER_GROUP`
  - 可选固定/限制/风场配置
- `MESH_CLOTH`
  - 引用 source mesh / proxy mesh / selection / mapping 规则
  - 关联多个 `COLLIDER_GROUP`
  - 关联 `CACHE_OUTPUT`

### 6. 数据模型设计

#### 6.1 Authoring 数据

- SpringBone authoring
  - armature
  - roots / joints
  - per-joint 参数覆盖
  - collider groups
  - center / inertia / gravity / drag
- ClothBone authoring
  - armature
  - root bones
  - 连接模式
  - 固定点、限制、形状保持参数
  - collider groups
- MeshCloth authoring
  - source render meshes
  - proxy mesh 生成规则
  - selection / weight / paint 来源
  - reduction / mapping 参数
  - cache 输出设置

#### 6.2 Compiled 数据

- `CompiledSpringJoint`
  - joint id
  - parent index
  - rest local transform
  - radius
  - stiffness / drag / gravity / limits
- `CompiledSpring`
  - spring id
  - center index
  - joint range
  - collider group indices
- `CompiledBoneCloth`
  - node list
  - edge list
  - bend pairs
  - fixed mask
  - bone writeback map
- `CompiledMeshCloth`
  - particle positions
  - edges / triangles
  - fixed mask / weights
  - mapping buffer
- `CompiledCollider`
  - shape type
  - transform
  - params
- `CompiledScene`
  - springs
  - cloth bones
  - mesh cloths
  - colliders
  - collider groups
  - cache descriptors

#### 6.3 Runtime 数据

- scene state
- solver buffers
- per-instance mutable settings
- debug / profiling counters
- export state

### 7. XPBD 求解器目标结构

#### 7.1 核心对象

- `SolverScene`
- `ParticleSet`
- `TransformSet`
- `ConstraintSet`
- `CollisionWorld`
- `MeshMappingSet`

#### 7.2 约束类型

- SpringBone
  - distance
  - angle limit
  - inertia
  - collider collision
- ClothBone
  - distance
  - bend / isometric bend
  - optional shape matching cluster
  - attachment / pin
- MeshCloth
  - stretch
  - bend
  - area / shape preserving
  - self collision
  - collider collision

#### 7.3 Step 流程

1. 写入 root / center / model 输入
2. 外力与惯性更新
3. fixed-step 积分（自动步数）
4. 碰撞体更新
5. 位置约束投影
6. 速度更新
7. 骨或网格结果输出

---

### 8. Blender UI 与交互目标

- UI 风格以 `bonex` 为主要借鉴对象：
  - 顶层运行面板
  - 子面板分组
  - 选中对象驱动参数面板
  - preset 入口
  - playback / bake / tool / debug 分区
- 交互原则：
  - 参数面板显示当前激活组件或当前选择骨
  - 运行状态与编辑状态分离
  - modal 工具只处理高价值交互
  - handler 只做播放联动和结果回写
  - 所有重结构改动显式提示 `Rebuild Required`

### 11. 分阶段路线

#### 第一阶段：SpringBone 正式化

- 从当前 `BONE_CHAIN` 升级为真正的 `SPRING_BONE`
- 引入 `COLLIDER_GROUP`
- 引入 per-joint 参数
- 原生 XPBD distance + angle + collision 跑通
- Blender live playback 稳定

#### 第二阶段：ClothBone

- 支持多 root 骨集合
- 构建骨网络拓扑
- 支持 bend / shape preserving
- 骨姿态批量回写
- 形成衣摆、披风、发束面状骨布料

#### 第三阶段：MeshCloth

- 代理网格导入/构建
- reduction / selection / mapping
- proxy 求解 + render 映射
- 低模实时 + 高模缓存输出

#### 第四阶段：工作流完善

- 预设系统
- 批量生成工具
- 视口调试
- bake / cache / 调试导出
- 性能统计与错误诊断

---

### 12. 当前优先事项

1. 把当前 MVP 的 `BONE_CHAIN` 方案提升为面向 `SPRING_BONE` 的组件结构。
2. 明确 `Collider / ColliderGroup / Spring / Joint` 四层数据模型。
3. 让 `compile -> build -> step -> pose apply` 适配未来多功能 compiled scene，而不是只适配骨链。
4. 为 `ClothBone` 和 `MeshCloth` 预留 compiled data、runtime API 和 UI 面板接口。
