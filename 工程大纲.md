## HoCloth 工程大纲

### 1. 项目定位

- **目标产品**：Blender 内可实时预览、可离线烘焙、可导出缓存的角色二级动力学插件。
- **核心路线**：`Blender/Python authoring + C++ native runtime`，总体目标从“写一个自己的物理内核”转为“把 `Magica Cloth 2` 做进 Blender”，交互流程继续以 `Bonex` 为主要参考。
- **目标对标**：
  - 物理内核方向以 `MC2` 为直接落地目标，而不再停留在“参考后自行实现”
  - UI / 交互 / 工作流对齐 `blender_bonex`
  - 当前优先实现 `MC2` 的 `BoneChain`
  - `referenceproject/` 内保存两套参考源码，用于逐项对照实现
- **当前重构策略**：
  - 本轮 native 侧按“彻底重构”处理，优先直接翻译 `Magica Cloth 2` 的关键 C# 源代码到 C++
  - 拿来使用的代码块必须以注释明确标明“来自 MC2”
  - 当前阶段不主动做逻辑层面的再设计或抽象优化，优先保证行为对齐；允许为移植可运行性和明确 bug 做最小修正
- **明确不做**：
  - 不以通用物理 SDK 作为主线方案
  - 不以 Blender 原生刚体系统作为核心求解器
  - 不把 Blender 自定义属性直接当作 runtime 输入结构

---

### 2. 产品主线

#### 2.1 BoneChain

- 当前主实现对象，目标是尽量贴近 `Magica Cloth 2` 的 `BoneChain`
- 典型对象：头发、尾巴、挂件链、裙摆骨、耳朵骨
- 已在推进的方向：
  - 单链或多链组织
  - 每 joint 独立参数接口
  - 中心骨、惯性、阻尼、重力、拖拽等基础控制
  - 基础碰撞与实时预览
- 当前状态：
  - 已完成不少参数接口
  - 已有最基础的物理效果，方向正确
  - 还需要继续深入照抄 `MC2` 的具体实现细节

#### 2.2 ClothBone

- 目标还原 MC2 的骨驱动布料。
- 本质不是“只有一条链”，而是“多根骨或骨网络的布料化求解 + 回写骨骼”。
- 关键能力：
  - 多 root 骨集合
  - 骨间连接拓扑构建
  - 距离 / 弯曲 / 形状保持约束
  - 骨姿态回写
  - 与碰撞体、风场、固定点联动

#### 2.3 MeshCloth

- 目标还原 MC2 的代理网格布料。
- 关键能力：
  - 从渲染网格或低模代理网格构建 simulation mesh
  - 代理网格 reduction / weld / selection / mapping
  - simulation mesh -> render mesh 映射
  - 实时低模预览 + 高模缓存输出

### 3. 总体架构原则

- **Authoring / Compiled / Runtime 三态分离**
  - Authoring：Blender 里编辑的组件与参数
  - Compiled：稳定、可导出、可调试的中间数据
  - Runtime：native 持有的 scene / solver / buffer / state
- **结构变更统一走 Rebuild**
  - 组件增删改
  - 骨拓扑变化
  - 碰撞体结构变化
  - 网格拓扑变化
- **每帧只传高频输入**
  - `dt`
  - 根节点 / 中心骨变换
  - 外力、风、模型级开关
  - 少量 joint / model 覆盖参数
- **求解器只接收扁平化结果**
  - 不向 C++ 传 Blender 对象
  - 不向 Python 暴露底层 solver 内部对象
  - Python 只看到 `handle + arrays + dict-like config`
- **实时主线优先骨回写，离线主线优先缓存**
  - BoneChain / ClothBone 重点是骨结果实时回写
  - MeshCloth 高模结果优先走缓存导出

### 4. 目标分层

- `authoring/`

  - debug dump

### 5. 组件系统目标

#### 5.1 一级组件
- `SPRING_BONE`
- `CLOTH_BONE`
- `MESH_CLOTH`
- `COLLIDER`
- `COLLIDER_GROUP`
- `WIND`
- `PIN` / `ATTACHMENT`
- `CACHE_OUTPUT`
- `PRESET`

#### 5.2 组件组织原则

- 保留主组件目录 + 类型容器的双层结构。
- 主组件只负责：
  - `component_id`
  - `component_type`
  - `container_index`
  - `display_name`
  - `enabled`
  - `ui_expanded`
- 类型容器负责：
  - 实际 authoring 数据
  - 类型特有参数
  - 与 Blender 对象的引用关系

#### 5.3 组件装配关系

- `SPRING_BONE`
  - 引用一组 `joint settings`
  - 关联多个 `COLLIDER_GROUP`
- `CLOTH_BONE`
  - 引用多个 root 骨
  - 引用拓扑与约束组
- `MESH_CLOTH`
  - 引用 render mesh / proxy mesh / skin data
  - 引用 collider / wind / cache settings

---

### 6. Compiled 数据目标

#### 6.1 需要编译的数据
- scene graph
- component summaries
- topology / adjacency
- collider sets
- solver buffers
- per-instance mutable settings
- debug / profiling counters
- export state

#### 6.2 目标原则
- 编译结果必须独立于 Blender 对象生命周期。
- 编译结果必须可序列化、可重建、可调试。
- 编译结果必须允许后续扩展到 ClothBone / MeshCloth。

---

### 7. 求解器目标结构

#### 7.1 核心对象
- `SolverScene`
- `ParticleSet`
- `TransformSet`
- `ConstraintSet`
- `CollisionWorld`
- `MeshMappingSet`

#### 7.2 约束类型
- BoneChain
  - distance
  - angle limit
  - inertia
  - collider collision
- ClothBone
  - distance
  - bend / isometric bend
  - optional shape matching cluster
  - attachment / pin
- MeshCloth
  - stretch
  - bend
  - area / shape preserving
  - self collision
  - collider collision

#### 7.3 Step 流程
1. 写入 root / center / model 输入
2. 外力与惯性更新
3. fixed-step 积分
4. 碰撞体更新
5. 位置约束投影
6. 速度更新
7. 骨或网格结果输出

---

### 8. Blender UI 与交互目标

- UI 风格以 `bonex` 为主要借鉴对象：
  - 顶层运行面板
  - 子面板分组
  - 选中对象驱动参数面板
  - preset 入口
  - playback / bake / tool / debug 分区
- 交互原则：
  - 参数面板显示当前激活组件或当前选择骨
  - 运行状态与编辑状态分离
  - modal 工具只处理高价值交互
  - handler 只做播放联动和结果回写
  - 所有重结构改动显式提示 `Rebuild Required`

### 11. 分阶段路线

#### 第一阶段：BoneChain 正式化
- 第一目标从“自研 BoneChain”转为“把 `MC2 BoneChain` 落进 Blender”
- 当前已清空既有 C++ 实现，native 侧按重新翻译 `MC2` 关键源码的方式重建
- 优先直接把 `Magica Cloth 2` 的关键 C# 逻辑逐段翻译为 C++，并尽量保持原始结构、命名与行为对应关系
- 所有直接移植或近似直译的实现块，都要在代码注释中标明来自 `MC2`
- 不以抽象成通用弹簧骨为优先，不主动做逻辑改写；只在发现明确问题时做必要修 bug
- 在此基础上再补齐 Blender 侧参数接口、碰撞组织、编译链路和播放稳定性

#### 第二阶段：ClothBone
- 支持多 root 骨集合
- 构建骨网络拓扑
- 支持 bend / shape preserving
- 骨姿态批量回写
- 形成衣摆、披风、发束面状骨布料

#### 第三阶段：MeshCloth
- 代理网格导入 / 构建
- reduction / selection / mapping
- proxy 求解 + render 映射
- 低模实时 + 高模缓存输出

#### 第四阶段：工作流完善
- 预设系统
- 批量生成工具
- 视口调试
- bake / cache / 调试导出
- 性能统计与错误诊断

---

### 12. 当前优先事项

1. 把当前目标明确切换为“将 `MC2` 做进 Blender”，而不是继续扩展自研 MVP 内核
2. 以 `MC2 BoneChain` 为第一落地点，直接翻译关键 C# 源码到 C++，并用注释标明来自 `MC2`
3. 在不主动做逻辑改写的前提下，补齐 `BoneChain` 相关参数接口、碰撞组织和行为对齐
4. 让 `compile -> build -> step -> pose apply` 适配这种“MC2 行为优先”的重构路线，并继续为 `ClothBone` 和 `MeshCloth` 预留 compiled data、runtime API 和 UI 面板接口
